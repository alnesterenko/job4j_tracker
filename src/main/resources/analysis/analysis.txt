Использовался инструмент jconsole.

1) При добавлении 1_000_000 Item-ов(заявок) в базу данных, расходуемая память постоянно "прыгала" 
 практически от 0 до 80 МБ.

2) При выводе в консоль 1_000_000 заявок, расходуемая память единоразово "скакнула" до 500 МБ.

3) При удалении 1_000_000 заявок, расходуемая память "скакала" в промежутке от 400 до 700 МБ.

Из сводки стало известно, что по умолчанию используется сборщик мусора G1.




Чтобы добиться выхода за пределы памяти, решил ограничить доступную память для G1 40МегаБайтами.

1) При добавлении 1_000_000 Item-ов(заявок) в базу данных, расходуемая память постоянно "прыгала" 
 практически от 7 до 30 МБ. 

2) При попытке удалить 1_000_000 заявок, программа(StartUI) завершилась ошибкой:
Exception in thread "main" java.lang.OutOfMemoryError: Java heap space
